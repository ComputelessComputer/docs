Anarlog includes a built-in **MCP (Model Context Protocol) server** that lets AI coding assistants and chat interfaces access your meeting notes as live context. Instead of copying and pasting from your notes into a chat window, you can ask your AI assistant a question and it will reach into your Anarlog vault to find the answer.

For example, you can ask Claude: *"What did we decide about the API design in last Tuesday's engineering sync?"* — and it will search your Anarlog notes and surface the relevant content directly in the conversation.

<Info>
  MCP is an open protocol developed by Anthropic that standardizes how AI assistants connect to external data sources and tools. Learn more at [modelcontextprotocol.io](https://modelcontextprotocol.io).
</Info>

## What the MCP server exposes

Once an AI assistant is connected to Anarlog's MCP server, it can:

- **Search your meeting notes** by keywords, date, participants, or topic
- **Read full note content** including structured sections and inline text
- **Access transcripts** linked to a meeting note
- **Retrieve summaries** and AI-enhanced content generated by Anarlog

The server does not expose your audio recordings or any data outside your Anarlog vault.

## Compatible AI clients

Anarlog's MCP server works with any MCP-compatible AI assistant, including:

- [Claude Desktop](https://claude.ai/download) (Anthropic)
- [Cursor](https://www.cursor.com/)
- [Windsurf](https://codeium.com/windsurf)
- Any other client that implements the Model Context Protocol

## Enabling the MCP server

Open **Settings → Intelligence** (or look for an **MCP** section in your Anarlog settings) and toggle the MCP server on. Anarlog will start the local server and display the connection details — including the local endpoint URL — that you'll paste into your AI client's configuration.

## Connecting Claude Desktop

To connect Claude Desktop to Anarlog, add an entry to Claude's MCP configuration file.

<Steps>
  <Step title="Locate Claude Desktop's config file">
    On macOS, the configuration file is at:

    ```
    ~/Library/Application Support/Claude/claude_desktop_config.json
    ```

    If the file doesn't exist yet, create it.
  </Step>
  <Step title="Add the Anarlog MCP server entry">
    Add the following JSON to the `mcpServers` object. The exact path to the `anarlog-mcp` binary depends on how you installed Anarlog — check the MCP settings panel in Anarlog for the confirmed path.

    ```json
    {
      "mcpServers": {
        "anarlog": {
          "command": "/Applications/Anarlog.app/Contents/MacOS/anarlog-mcp",
          "args": []
        }
      }
    }
    ```

    If you already have other MCP servers configured, add the `"anarlog"` entry alongside them inside the existing `mcpServers` object.
  </Step>
  <Step title="Restart Claude Desktop">
    Save the file and restart Claude Desktop. The Anarlog integration will appear in Claude's available tools list.
  </Step>
  <Step title="Test the connection">
    In a new Claude conversation, ask something like: *"Search my Anarlog notes for mentions of [topic]."* Claude will call the Anarlog MCP tool and return results from your vault.
  </Step>
</Steps>

## Connecting Cursor or Windsurf

Cursor and Windsurf follow the same MCP configuration pattern. In each application's settings, look for an **MCP Servers** or **Tools** section and add a new server entry pointing to the `anarlog-mcp` binary with the path shown in Anarlog's MCP settings panel.

Refer to each application's documentation for the exact location of its MCP configuration file or UI.

<Note>
  The Anarlog MCP server runs entirely on your Mac. Your meeting notes are never sent to a third-party server by Anarlog itself. Keep in mind, however, that when an AI assistant queries your notes, that content is included in the context sent to the AI provider (such as Anthropic's API for Claude). Review your AI provider's privacy policy if this is a concern.
</Note>
